About Go Server

Go Server is a CLI tool. The CLI compiles a derivative of xml (.gxml file extension, Go server markup) to build web applications. Go server is the agile tool for you. The markup was designed to offer a cleaner work canvas to developers. Visit the Go server markup markup to learn more about the behavior of xml tags.

Syntax

This CLI converts the xml tags within your Go Server file into Go lang code. Find more information about the behavior of tags in the Go server markup section. Get the autocomplete for sublime 3, Search for Gopher Sauce on sublime package control.

Extensions

Any of your declared tags within a Go Server xml file can be exported to another project. Checkout the active MVC via websocket project. To have a Go Server extension listed create a new issue with a link to your repository or update the extensionsâ€™ page wiki.

How to install GoS

go get github.com/cheikhshift/gos
go install github.com/cheikhshift/gos
gos deps

Dependencies

The application generated by Go Server has certain dependencies. Please run the command below to download them before attempting to run or export your application. (Assuming $GOPATH is set)

$GOPATH/bin/gos deps

Quick Start your project

mkdir new-project
    cd new-project
    gos --make

How to debug

Please use current golang command line tools as a fallback. The final result of each Go server webapp is go code.

Go Server apps

  • Find bugs by reading your web application's output. If you're writing a webapp, Go Server attempts to use logging to find bugs. This process works by recovering from a runtime panic and stating the defunct line including the reason it crashed. This logger is only supported within the web service code, template pipeline code and templates. Due to this feature limit your code within the scope of one line.

Good :

var name := Struct{Property:"val"}

Bad for debug logger :

var name := Struct{ Property:"val"
		    FieldTwo: 2
		   }
