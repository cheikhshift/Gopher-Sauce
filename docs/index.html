About Go Server

Go Server is a CLI tool. The CLI reads a derivative of xml (.gxml) to build a web application.

Project structure :

		
folder/
	tmpl/
	web/
	gos.gxml
	...additonal .go files too with package main

gos.gxml — File with Golang server markup.

web/ — Project's web root.

tmpl/ — Project's template root. Read more about templates here

XML Syntax

Your GO code is wrapped around XML tags. Check out the webservice project to visualize the placement of GO Code within XML. Find more information about the behavior of tags in the Go server markup section. Get the autocomplete for sublime 3, Search for Gopher Sauce on sublime package control.

How to debug

Please use current golang command line tools as a fallback. The final result of each Go server webapp is go code.

Go Server apps

  • Find bugs by reading your web application's output. If you're writing a webapp, Go Server attempts to use logging to find bugs. This process works by recovering from a runtime panic and stating the defunct line including the reason it crashed. This logger is only supported within the web service code, template pipeline code and templates. Due to this feature limit your code within the scope of one line.

Good :

var name := Struct{Property:"val"}

Bad for debug logger :

var name := Struct{ 
		    Property:"val",
		    FieldTwo: 2
		   }
