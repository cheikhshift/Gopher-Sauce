    HTTP 1.1 services

    HTTP 1.1 handlers are declared in GXML with tag <end/>.

    Syntax

    End tags are placed at the root of your <endpoints/> tag.

    GXML attributes :

    • path : request path to handle.

    • type : specifies request type to handle, can be HTTP 1.1 RFC request methods (GET, PUT …etc), star or f.

      • type f end tags are executed if the request path includes the path specified.
      • type star end tags handle all requests matching path attribute, disregarding any passed request methods.

    • id : specifies ID to use with opentracing.

    GXML content :

    Defines Go code to be ran on handler execution.

    Comments

    To comment an HTTP handler use XML style comments. The comments must be in your tag’s GXML content. For example :

     <end>
 <!-- Comment -->
 </end>
    Scope of code

    You may use other Go packages without explicitly importing them. This is due to the fact that gos uses a command to add missing imports to generated source code.

    Variables
    • var response string : JSON string to return as response.
    • var callmet bool : If set to true prevents the next handler from executing.
    • var r *http.Request : Current request to handler.
    • var w http.ResponseWriter : Response of request.
    • var session *"github.com/gorilla/sessions".Session : Current user session.
    Debug variables

    These variables are only available on debug :

    • var span opentracing.Span : Span of current request.
    Functions

    These functions are available to your code :

    • func mResponse(v interface{}) string : Converts a struct into a formatted string. For example JSON, XML or YAML.

    Sample tag for root of web server :

    <endpoints>
	<end path="/" id="foo" type="GET" >
		<!-- Handler comment -->
  		response = "Hi"
  	</end>
 </endpoints>
